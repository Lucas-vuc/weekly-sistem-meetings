1 · The core idea in one sentence

The meeting is the database. What we decide in each meeting is written directly into a shared workspace; the next meeting reads from it; no export, no import, no Jira ticket creation step in between.

2 · Six meetings — one continuous loop

The system runs on six meetings. Each one consumes the output of the previous and produces the input for the next.

5 · The three permanent monitors

Open any time. Not meetings — they're pages that any founder can check between meetings.

Portfolio Matrix

Quick Kill Filter (6 gates) + 8-dimension scoring + decision ladder (Disregard · Park · Queue · Pursue · Execute now). The full universe of projects with current scores. Sourced from the May 04 Monthly Strategic.

Top 5 Live

Current ranking. Scores editable any time with 2-founder ratification. Audit log of every change. Bi-monthly ratifies accumulated changes formally.

KPI Dashboard

All tracked KPIs (VUC + VWC) with target vs actual, trend, alerts. Feeds Monthly Part 1 directly. Initial KPI list to be defined once meeting data accumulates.

7 · What changes from today

What goes away

  • Jira tickets — no more double registration of work
  • JSON sneakernet — no export → DM → import workflow
  • HTML files as the working tool — replaced by Notion pages (kept as archive)
  • Re-typing objectives between meetings

What we gain

  • Real-time multi-user editing — see each other's changes live
  • Permanent Portfolio & Top 5 views — not only during Bi-monthly
  • Mobile — Notion app for on-site work (critical for Emmanuel)
  • Linked objectives — weekly task → monthly objective → Top 5 → progress rolls up
  • Search, history, comments, mentions across all data

9 · Implementation path

  1. Founder buy-in (this document) — confirm we move forward
  2. Workspace creation — admin sets up Notion Team workspace, invites the 4 founders, sets Emmanuel restricted access
  3. Schema build (2–3 hour workshop) — create the 7 databases with their properties
  4. Initial seed — historical data loaded from existing JSONs, PDFs, and Jira export. No starting from zero.
  5. Walk-through (1 hour) — open each meeting page, simulate a week
  6. Go-live week 1 — parallel run with Jira
  7. Sunset Jira — end of week 1 if no major friction
